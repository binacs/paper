# Hyperledger fabric: a distributed operating system for permissioned blockchains

## TL;DR

Hyperledger Fabric is a modular, permissioned blockchain platform that supports general-purpose smart contracts without a native cryptocurrency, achieving over 3500 transactions per second with sub-second latency.

## Problem & Motivation

Existing permissioned blockchain platforms are often rigid, requiring domain-specific languages for smart contracts and relying on native cryptocurrencies. They also struggle with non-determinism, resource exhaustion, and performance attacks. Fabric addresses these issues by providing a flexible, modular architecture that decouples consensus, membership, and execution.

## Key Ideas

- Modular consensus protocols allowing customization for different trust models.
- Smart contracts in standard programming languages (e.g., Go, Java) without a native cryptocurrency.
- Portable membership model integrated with industry-standard identity management.
- Novel blockchain design that handles non-determinism and resource exhaustion.
- Execute-order-validate architecture separating transaction execution from ordering.

## System Design

Fabric consists of peers (endorsers and committers), ordering service (consensus), and membership service provider (MSP). Clients submit proposals to endorsing peers, which simulate execution. Endorsed proposals are sent to the ordering service, which orders transactions into blocks. Blocks are delivered to committing peers for validation and ledger update. The system supports multiple channels for data isolation.

## Evaluation

The paper evaluates Fabric by implementing a Bitcoin-inspired digital currency. Experiments show end-to-end throughput over 3500 transactions per second in popular deployment configurations, with sub-second latency, scaling to over 100 peers.

## Takeaways

- Modular design enables tailoring to specific use cases and trust models.
- Support for general-purpose languages lowers the barrier for developing distributed applications.
- No native cryptocurrency reduces complexity and regulatory concerns.
- Execute-order-validate architecture improves performance and flexibility.
- Membership portability allows integration with existing identity systems.

## Related Work

Fabric contrasts with existing blockchain platforms like Ethereum and Bitcoin that require domain-specific languages and rely on cryptocurrencies. It is the first extensible permissioned blockchain system to support general-purpose smart contracts.

## Open Questions

Performance under adversarial conditions or with complex smart contracts is not fully explored. The paper does not address cross-channel interoperability or scalability beyond 100 peers. Future work could investigate dynamic membership changes and integration with other Hyperledger projects.
