# Hyperledger fabric: a distributed operating system for permissioned blockchains

## TL;DR

Hyperledger Fabric is a modular, permissioned blockchain platform that supports general-purpose smart contracts without a native cryptocurrency, achieving over 3500 transactions per second with sub-second latency.

## Problem & Motivation

Existing blockchain platforms like Bitcoin and Ethereum are permissionless, rely on native cryptocurrencies, and require smart contracts to be written in domain-specific languages. They suffer from non-determinism, resource exhaustion, and performance attacks. Permissioned blockchains need to support modular consensus, identity management, and general-purpose programming languages for enterprise use cases.

## Key Ideas

- Modular consensus protocols that can be tailored to specific trust models.
- Smart contracts in standard programming languages (e.g., Go, Java) without a native cryptocurrency.
- Portable membership model integrated with industry-standard identity management.
- Novel blockchain design addressing non-determinism, resource exhaustion, and performance attacks.

## System Design

Fabric is a distributed operating system for permissioned blockchains. It consists of peers (endorsers and committers), ordering service (consensus), and membership service provider (identity). Transactions are executed in three phases: endorsement (simulation), ordering (consensus), and validation (commit). The system supports pluggable consensus (e.g., Kafka, Raft) and channels for private data.

## Evaluation

The paper evaluates Fabric by implementing a Bitcoin-inspired digital currency. It achieves end-to-end throughput of over 3500 transactions per second in popular deployment configurations, with sub-second latency, scaling to over 100 peers.

## Takeaways

- Fabric is the first extensible blockchain system for permissioned networks with general-purpose smart contracts.
- Modular consensus and membership enable flexibility for diverse enterprise use cases.
- The three-phase transaction flow (endorse-order-validate) decouples execution from ordering, improving performance and scalability.

## Related Work

Fabric contrasts with permissionless blockchains like Bitcoin and Ethereum that require native cryptocurrencies and domain-specific smart contract languages. It also differs from other permissioned platforms by supporting general-purpose languages and modular consensus.

## Open Questions

Limitations include reliance on a centralized ordering service (though pluggable), potential performance bottlenecks in endorsement phase, and complexity of channel management. Future work may explore decentralized ordering, improved scalability, and integration with other Hyperledger projects.
