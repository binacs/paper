# AAFLOW: Scalable Patterns for Agentic AI Workflows

## TL;DR

AAFLOW is a unified distributed runtime that models agentic LLM workflows as an operator abstraction, using Apache Arrow and Cylon for zero-copy data transfer and resource-deterministic scheduling to achieve up to 4.64× pipeline speedup and 2.8× gains in embedding/upsert phases without LLM inference acceleration.

## Problem & Motivation

Existing agentic LLM frameworks suffer from scalability and reproducibility limitations due to fragmented data orchestration, serialization overhead, and non-deterministic execution. They lack a formal execution model adhering to high-performance computing principles, leading to inefficiencies in data flow and coordination.

## Key Ideas

- Model agentic workflows as an operator abstraction for communication-efficient execution plans.
- Use Apache Arrow and Cylon to create a zero-copy data plane, eliminating serialization overhead between preprocessing, embedding, and vector retrieval.
- Employ resource-deterministic scheduling and asynchronous batching to lower coordination costs.

## System Design

AAFLOW is a unified distributed runtime. It structures workflows as operators that are scheduled with resource-deterministic scheduling. Data flows through a zero-copy data plane built on Apache Arrow and Cylon, enabling direct interoperability between components without serialization. Asynchronous batching is used to improve throughput.

## Evaluation

Experiments show up to 4.64× pipeline speedup and 2.8× gains in embedding and upsert phases compared to baselines, while retaining comparable LLM generation throughput. The improvements come from enhanced data flow, batching, and communication efficiency, not LLM inference acceleration.

## Takeaways

- Agentic workflow performance can be significantly improved by optimizing data orchestration and communication, not just LLM inference.
- Zero-copy data planes (Arrow/Cylon) eliminate serialization bottlenecks.
- Resource-deterministic scheduling and asynchronous batching reduce coordination overhead.

## Related Work

AAFLOW addresses limitations of existing agentic LLM frameworks that increase flexibility but lack formal execution models and suffer from scalability and reproducibility issues. It contrasts with systems that do not optimize data flow and communication.

## Open Questions

How does AAFLOW handle dynamic or heterogeneous agentic workflows? What are the trade-offs of resource-deterministic scheduling in highly variable workloads? Can the approach be extended to other data-intensive AI pipelines beyond LLM agents?
