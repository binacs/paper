# Learning scheduling algorithms for data processing clusters

## TL;DR

Decima uses reinforcement learning and neural networks to automatically learn workload-specific scheduling policies for data processing clusters, outperforming hand-tuned heuristics by at least 21% in average job completion time.

## Problem & Motivation

Efficient scheduling of data processing jobs on distributed clusters is critical but complex. Current systems rely on simple, generalized heuristics that ignore workload characteristics, as developing and tuning a custom scheduling policy for each workload is infeasible. This leads to suboptimal performance, especially under high load.

## Key Ideas

- Use reinforcement learning (RL) to automatically generate scheduling policies without human instruction beyond a high-level objective (e.g., minimizing average job completion time).
- Develop new representations for jobs' dependency graphs to capture complex structures.
- Design scalable RL models that can handle the complexity and scale of the scheduling problem.
- Invent RL training methods for dealing with continuous stochastic job arrivals.

## System Design

Decima is an RL-based scheduler integrated with Spark. It uses neural networks to represent job dependency graphs and makes scheduling decisions (e.g., which task to run next) based on the learned policy. The system handles continuous job arrivals by training with stochastic arrivals. The control flow involves observing the cluster state, selecting actions via the RL policy, and updating the policy based on rewards (e.g., job completion times).

## Evaluation

Evaluated on a 25-node Spark cluster. Baselines include hand-tuned scheduling heuristics. Headline results: Decima improves average job completion time by at least 21%, achieving up to 2x improvement during periods of high cluster load.

## Takeaways

- Machine learning can automatically generate efficient scheduling policies tailored to workload characteristics.
- Off-the-shelf RL techniques are insufficient; custom representations and training methods are needed for scheduling.
- Decima's approach significantly outperforms hand-tuned heuristics, especially under high load.

## Related Work

Positioned against traditional scheduling heuristics used in systems like Spark and Hadoop. Related work includes prior attempts at workload-aware scheduling and RL for resource management, but Decima is the first to learn scheduling algorithms end-to-end for data processing clusters.

## Open Questions

- How does Decima scale to much larger clusters (e.g., thousands of nodes)?
- Can the learned policies generalize to unseen workloads or cluster configurations?
- What is the overhead of the RL model inference and training in production settings?
