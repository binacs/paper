# CvxCluster: Solving Large, Complex, Granular Resource Allocation Problems 100-1000x Faster

## TL;DR

CvxCluster uses convex optimization to solve cluster resource allocation problems 100-2500x faster than MIP solvers, with solutions within 3% of optimal, by reformulating placement as a continuous problem.

## Problem & Motivation

Cluster resource allocation is a multidimensional search problem that finds the best allocation of tasks to servers. The search space grows exponentially, so modern approaches use mixed integer programming (MIP) or complex heuristics, which are slow and struggle with large, complex, granular constraints.

## Key Ideas

- Reformulate cluster resource allocation as a convex optimization problem, enabling extremely fast solution methods.
- Two-stage algorithm: first stage solves a convex relaxation to produce per-machine resource prices; second stage uses these prices to drive a lightweight greedy placement procedure.
- The convex relaxation provides principled prices that guide the greedy heuristic to near-optimal placements.

## System Design

CvxCluster has two stages: (1) Solve a convex relaxation of the placement problem to compute per-machine resource prices. (2) Use these prices in a lightweight greedy algorithm to place tasks onto servers. The convex relaxation is continuous, avoiding the combinatorial explosion of discrete search.

## Evaluation

Evaluated with Azure traces. Scales to 100,480 servers under proportional workload growth. Sustains arrival rates up to 500,000x baseline trace. Runs 100-2500x faster than a state-of-the-art MIP solver while staying within 3% of optimal objective. Supports complex constraints like job anti-affinity, machine types, and GPU servers.

## Takeaways

- Convex optimization can dramatically speed up cluster resource allocation without sacrificing solution quality.
- A two-stage approach (convex relaxation + greedy) is effective for large-scale placement.
- The method handles complex constraints and scales to hundreds of thousands of servers.

## Related Work

Contrasts with MIP solvers and heuristic search methods; shows that convex optimization can outperform both in speed and solution quality.

## Open Questions

How does the approach perform with highly heterogeneous workloads or dynamic resource demands? Can the convex relaxation be extended to handle online or streaming placement? What are the limits of the greedy stage when constraints become very tight?
