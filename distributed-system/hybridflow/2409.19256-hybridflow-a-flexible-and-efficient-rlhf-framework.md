# HybridFlow: A Flexible and Efficient RLHF Framework

## TL;DR

HybridFlow combines single-controller and multi-controller paradigms to flexibly represent and efficiently execute the complex dataflow of RLHF, achieving 1.53x–20.57x throughput improvement over state-of-the-art baselines.

## Problem & Motivation

Reinforcement Learning from Human Feedback (RLHF) for LLM alignment involves a complex dataflow where each node is a distributed LLM training or generation program and each edge is a many-to-many multicast. Traditional RL frameworks use a single controller to orchestrate both intra-node computation and inter-node communication, but this leads to large control dispatch overhead for distributed intra-node computation. Existing RLHF systems adopt a multi-controller paradigm, which is inflexible due to nesting distributed computation and data communication. There is a need for a framework that combines the efficiency of single-controller with the flexibility of multi-controller to handle RLHF's unique challenges.

## Key Ideas

- Hybrid single-controller and multi-controller paradigm for flexible representation and efficient execution of RLHF dataflow.
- Hierarchical APIs that decouple and encapsulate computation and data dependencies, enabling efficient operation orchestration and flexible device mapping.
- 3D-HybridEngine for efficient actor model resharding between training and generation phases, achieving zero memory redundancy and reduced communication overhead.

## System Design

HybridFlow uses a hybrid controller design: a single global controller manages high-level dataflow orchestration, while multiple local controllers handle intra-node distributed computation. The hierarchical APIs separate computation (neural network operations) from data dependencies (communication patterns). The 3D-HybridEngine supports resharding of the actor model across three dimensions (data, tensor, pipeline parallelism) between training and generation phases, with zero memory redundancy.

## Evaluation

The paper evaluates HybridFlow on various RLHF algorithms (e.g., PPO, GRPO) and compares with state-of-the-art baselines (e.g., DeepSpeed-Chat, OpenRLHF). Headline results show 1.53x–20.57x throughput improvement. Workloads include LLMs of different sizes (e.g., 7B, 13B, 70B parameters).

## Takeaways

- Hybrid controller paradigm effectively balances flexibility and efficiency for RLHF dataflow.
- Decoupling computation and data dependencies via hierarchical APIs simplifies implementation of complex RLHF algorithms.
- 3D-HybridEngine enables efficient model resharding with zero memory redundancy, critical for alternating training and generation phases.
- Significant throughput gains over existing systems demonstrate practical impact.

## Related Work

HybridFlow is positioned against traditional RL frameworks (single-controller) and existing RLHF systems (multi-controller). It improves upon DeepSpeed-Chat, OpenRLHF, and other baselines by combining both paradigms.

## Open Questions

Scalability to extremely large models (e.g., >100B parameters) and heterogeneous hardware. Generalization of the hybrid paradigm to other distributed training workflows beyond RLHF. Potential overhead of the hierarchical API abstraction in very large deployments.
