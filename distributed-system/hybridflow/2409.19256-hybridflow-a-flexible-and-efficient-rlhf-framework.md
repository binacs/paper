# HybridFlow: A Flexible and Efficient RLHF Framework

## TL;DR

HybridFlow combines single-controller and multi-controller paradigms to flexibly represent and efficiently execute the complex dataflow of RLHF, achieving 1.53x–20.57x throughput improvement over state-of-the-art baselines.

## Problem & Motivation

Reinforcement Learning from Human Feedback (RLHF) for LLM alignment involves a complex dataflow where each node is a distributed LLM training or generation program and each edge is a many-to-many multicast. Traditional RL frameworks use a single controller for both intra-node computation and inter-node communication, leading to high control dispatch overhead for distributed intra-node computation. Existing RLHF systems adopt a multi-controller paradigm, but this is inflexible due to nesting distributed computation and data communication.

## Key Ideas

- Hybrid single-controller and multi-controller paradigm for flexible representation and efficient execution of RLHF dataflow.
- Hierarchical APIs that decouple and encapsulate computation and data dependencies, enabling efficient operation orchestration and flexible device mapping.
- 3D-HybridEngine for efficient actor model resharding between training and generation phases, achieving zero memory redundancy and reduced communication overhead.

## System Design

HybridFlow combines a single-controller for global orchestration and multi-controllers for local distributed execution. The hierarchical APIs allow users to define the RLHF dataflow by composing high-level operations that abstract away distributed details. The 3D-HybridEngine handles resharding of the actor model across training and generation phases, leveraging 3D parallelism (data, tensor, pipeline) to minimize memory and communication costs.

## Evaluation

Experiments running various RLHF algorithms show 1.53x–20.57x throughput improvement compared to state-of-the-art baselines. Workloads and baselines are not detailed in the abstract.

## Takeaways

- HybridFlow demonstrates that a hybrid control paradigm can outperform both pure single-controller and multi-controller approaches for RLHF.
- Decoupling computation and data dependencies via hierarchical APIs simplifies implementation of complex RLHF algorithms.
- Efficient model resharding between training and generation is critical for RLHF performance.

## Related Work

HybridFlow is positioned against traditional RL frameworks (single-controller) and existing RLHF systems (multi-controller), claiming to combine their advantages.
