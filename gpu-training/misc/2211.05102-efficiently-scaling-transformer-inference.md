# Efficiently Scaling Transformer Inference

## TL;DR

Develops a simple analytical model and low-level optimizations to select multi-dimensional partitioning for efficient Transformer inference on TPU v4, achieving new Pareto frontier on latency and MFU for 500B+ parameter models, and shows multiquery attention enables 32x longer contexts.

## Problem & Motivation

Generative inference for large Transformer models is challenging due to tight latency targets, long sequence lengths, and large model sizes. The engineering tradeoffs for efficient inference are not well understood, especially for models with 500B+ parameters. Existing solutions like FasterTransformer may not be optimal for all scenarios.

## Key Ideas

- Analytical model for inference efficiency to select optimal multi-dimensional partitioning techniques on TPU v4 slices.
- Suite of low-level optimizations to improve latency and model FLOPS utilization (MFU).
- Use of multiquery attention to reduce memory requirements, enabling scaling context lengths by 32x.
- Achieves low-batch-size latency of 29ms per token with int8 weight quantization and 76% MFU during large-batch processing.

## System Design

The system uses TPU v4 slices with multi-dimensional partitioning (e.g., data, model, pipeline parallelism) chosen via an analytical model based on application requirements. Low-level optimizations include int8 quantization and efficient attention implementations. Control flow is not detailed; data flow involves partitioning across TPU cores.

## Evaluation

Validated on PaLM 540B parameter model with 2048-token context. Baselines include FasterTransformer. Headline numbers: 29ms per token latency (low batch, int8), 76% MFU (large batch). Pareto frontier outperforms FasterTransformer on latency-MFU tradeoffs.

## Takeaways

- Analytical models can guide partitioning choices for inference.
- Multiquery attention is key for long contexts.
- Int8 quantization reduces latency without significant quality loss.
- TPU v4 slices can achieve high MFU for large models.

## Related Work

Positioned against FasterTransformer suite of benchmarks, showing improved Pareto frontier. Related to general Transformer inference optimization and multiquery attention.

## Open Questions

Generalizability to other hardware (e.g., GPUs) and model architectures. Impact of quantization on model quality. Scalability to even larger models or longer sequences beyond 2048 tokens.
