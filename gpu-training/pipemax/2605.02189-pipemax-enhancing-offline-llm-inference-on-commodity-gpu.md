# PipeMax: Enhancing Offline LLM Inference on Commodity GPU Servers

## TL;DR

PipeMax integrates pipeline parallelism with KV cache offloading to maximize offline LLM inference throughput on commodity GPU servers, achieving up to 2.51x higher throughput than vLLM.

## Problem & Motivation

Offline LLM inference aims to maximize request throughput under fixed budgets, making commodity GPU servers attractive. However, prior work treats offloading and parallelism separately, leading to suboptimal performance due to interconnect and memory constraints. Existing systems either use tensor parallelism (high communication overhead) or pipeline parallelism without effective memory expansion, limiting batch sizes and throughput.

## Key Ideas

- Integrate pipeline parallelism with KV cache offloading to overcome GPU memory limits.
- Pipeline parallelism keeps only one batch active per GPU, enabling offloading of inactive batches' KV caches.
- Coordinate computation with offloading data movement to hide latency and sustain large-batch execution.

## System Design

PipeMax uses pipeline parallelism across GPUs, where each GPU processes a different micro-batch in a pipeline stage. Only one batch is active on each GPU at a time; KV caches of inactive batches are offloaded to CPU memory or NVMe. The system schedules computation and offloading operations to overlap, reducing idle time and effectively expanding GPU memory capacity.

## Evaluation

Experiments on an 8-GPU node show PipeMax achieves up to 2.51x higher throughput than vLLM, and up to 1.42x and 1.38x higher throughput than state-of-the-art high-throughput LLM systems (likely FlexGen and DeepSpeed-Inference). Workloads include various LLMs and batch sizes.

## Takeaways

- Pipeline parallelism naturally enables KV cache offloading due to single active batch per GPU.
- Coordinating computation and offloading is key to hiding data movement latency.
- Commodity GPU servers can achieve high throughput with proper integration of parallelism and offloading.

## Related Work

PipeMax contrasts with prior work that considers offloading and parallelism in isolation. It builds on pipeline parallelism (e.g., GPipe) and offloading techniques (e.g., FlexGen), but uniquely combines them for offline inference.

## Open Questions

Scalability to larger GPU clusters (beyond 8 GPUs) and heterogeneous memory hierarchies. Impact of offloading on latency for real-time inference (not offline). Trade-offs between pipeline depth and offloading overhead.
