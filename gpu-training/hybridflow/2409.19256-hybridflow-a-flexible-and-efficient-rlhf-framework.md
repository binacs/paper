# HybridFlow: A Flexible and Efficient RLHF Framework

## TL;DR

HybridFlow combines single-controller and multi-controller paradigms to flexibly represent and efficiently execute the complex dataflow of RLHF, achieving 1.53x–20.57x throughput improvement over state-of-the-art baselines.

## Problem & Motivation

Reinforcement Learning from Human Feedback (RLHF) for LLM alignment involves a complex dataflow where each node is a distributed LLM training or generation program and each edge is a many-to-many multicast. Traditional RL frameworks use a single controller to orchestrate both intra-node computation and inter-node communication, but this leads to large control dispatch overhead for distributed intra-node computation. Existing RLHF systems adopt a multi-controller paradigm, which is inflexible due to nesting distributed computation and data communication. There is a need for a framework that combines the benefits of both paradigms to enable flexible representation and efficient execution.

## Key Ideas

- Hybrid single-controller and multi-controller paradigm for RLHF dataflow.
- Hierarchical APIs that decouple and encapsulate computation and data dependencies.
- 3D-HybridEngine for efficient actor model resharding between training and generation phases with zero memory redundancy and reduced communication overhead.

## System Design

HybridFlow uses a hybrid controller approach: a single controller handles global orchestration while multiple controllers manage distributed intra-node computation. The hierarchical APIs allow users to define computation and data dependencies separately, enabling flexible mapping onto devices. The 3D-HybridEngine handles resharding of the actor model between training and generation phases, leveraging 3D parallelism (data, tensor, pipeline) to minimize memory and communication overhead.

## Evaluation

The paper evaluates HybridFlow on various RLHF algorithms (e.g., PPO, GRPO) and compares against state-of-the-art baselines (e.g., DeepSpeed-Chat, OpenRLHF). Throughput improvements range from 1.53x to 20.57x. Workloads include LLMs of different sizes (e.g., 7B, 13B, 70B).

## Takeaways

- Hybrid controller paradigm effectively balances flexibility and efficiency for RLHF dataflow.
- Decoupling computation and data dependencies via hierarchical APIs simplifies algorithm implementation and device mapping.
- 3D-HybridEngine enables efficient model resharding with zero memory redundancy, critical for alternating training and generation phases.
- Significant throughput gains demonstrate practical impact for LLM alignment.

## Related Work

HybridFlow is positioned against traditional RL frameworks (single-controller) and existing RLHF systems (multi-controller). It improves upon DeepSpeed-Chat, OpenRLHF, and other baselines by combining both paradigms.

## Open Questions

Scalability to extremely large models (e.g., >100B parameters) and heterogeneous hardware. Generalization to other multi-model dataflow patterns beyond RLHF. Potential overhead of hybrid control in very large clusters.
