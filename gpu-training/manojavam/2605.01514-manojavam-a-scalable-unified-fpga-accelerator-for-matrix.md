# MANOJAVAM: A Scalable, Unified FPGA Accelerator for Matrix Multiplication and Singular Value Decomposition in Principal Component Analysis

## TL;DR

MANOJAVAM is a scalable FPGA accelerator that unifies matrix multiplication and SVD for PCA, achieving up to 22.75x speedup and 42.14x energy reduction over an NVIDIA A6000 GPU.

## Problem & Motivation

PCA is computationally intensive due to matrix multiplication and SVD bottlenecks. Prior hardware accelerators are either specialized for one stage, use HLS limiting microarchitectural optimizations, or employ fixed-point datapaths that limit scalability across different input dimensions. There is a need for a unified, scalable accelerator that can handle datasets of any size efficiently.

## Key Ideas

- Unifies matrix multiplication and SVD in a single architecture.
- Uses TPU-style systolic arrays (TxT) with block streaming for high-throughput matrix multiplication.
- Integrates a highly parallel Jacobian unit implementing the Jacobi method for SVD with pipelined CORDIC-based rotations.
- Employs a two-tier cache hierarchy and mode-aware memory policies to adapt to distinct memory access patterns of covariance matrix and rotation computation.

## System Design

MANOJAVAM(T,S) consists of S number of TxT systolic arrays for matrix multiplication. The Jacobian unit performs SVD using the Jacobi method with pipelined CORDIC rotations. A two-tier cache hierarchy and mode-aware memory policies manage data flow: one mode for covariance matrix computation and another for rotation updates. The architecture is parameterized by T (systolic array size) and S (number of arrays), enabling scalability.

## Evaluation

Two implementations: MANOJAVAM(4,8) on Xilinx Artix-7 FPGA at 200 MHz, 1.271W; MANOJAVAM(16,32) on Xilinx Virtex-Ultrascale+ at 434 MHz, 16.957W. Benchmarked on real-world datasets against NVIDIA A6000 GPU: up to 22.75x speedup in SVD latency and 42.14x reduction in total energy consumption.

## Takeaways

- Unifying matrix multiplication and SVD in one architecture avoids data movement between separate accelerators.
- Systolic arrays with block streaming provide high throughput for matrix multiplication.
- Jacobi method with pipelined CORDIC enables efficient SVD.
- Mode-aware memory policies optimize for different computation phases.
- The design is scalable across different T and S parameters, suitable for both edge and high-performance computing.

## Related Work

Prior PCA accelerators target only one stage (matrix multiplication or SVD) or use HLS/fixed-point datapaths. MANOJAVAM is a unified, scalable, fully custom architecture that handles arbitrary input dimensions.

## Open Questions

How does the architecture scale to larger T and S beyond 16 and 32? What is the performance on datasets with extreme dimensions? Could the Jacobian unit be replaced with a more efficient SVD algorithm? How does it compare to other FPGA-based PCA accelerators?
