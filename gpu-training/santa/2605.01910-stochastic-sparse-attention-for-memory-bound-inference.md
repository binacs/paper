# Stochastic Sparse Attention for Memory-Bound Inference

## TL;DR

SANTA sparsifies KV-cache access during autoregressive decoding by sampling a small subset of value vectors from the post-softmax distribution, yielding unbiased estimates and significant speedups without accuracy loss.

## Problem & Motivation

Autoregressive decoding at long contexts is memory-bound because generating each token requires reading all key and value vectors from the KV cache. This bandwidth limitation becomes severe as context length grows, making inference slow and energy-inefficient.

## Key Ideas

- Stochastic Additive No-mulT Attention (SANTA): sparsifies value-cache access by sampling S << n_k indices from the post-softmax distribution and aggregating only those value rows.
- Unbiased estimator of post-softmax value aggregation, replacing multiply-accumulates with gather-and-add.
- Stratified sampling for variance-reduced, GPU-friendly variants.
- Bernoulli qK^T sampling: sparsifies the score stage by stochastic ternary queries, reducing key-feature access.

## System Design

SANTA modifies the attention mechanism: during decoding, after computing softmax scores, it samples a small number of value indices instead of reading all values. The sampled values are gathered and added (no multiplication). Stratified sampling is used to improve variance. Bernoulli qK^T sampling further reduces key access by using ternary queries.

## Evaluation

Evaluated on an NVIDIA RTX 6000 Ada with 32k-token contexts. Achieves 1.5× decode-step attention kernel speedup over FlashInfer and FlashDecoding while matching baseline accuracy.

## Takeaways

- SANTA provides a principled way to sparsify attention without bias.
- Stratified sampling is key to variance reduction and GPU efficiency.
- The method is orthogonal to quantization, low-rank projections, and KV-cache compression.
- Points toward multiplier-free, energy-efficient inference.

## Related Work

Orthogonal to upstream techniques such as ternary quantization, low-rank projections, and KV-cache compression. Compared against FlashInfer and FlashDecoding.

## Open Questions

How does SANTA perform at even longer contexts (e.g., 128k+)? Can the sampling overhead be further reduced? How does it interact with other sparsity methods in practice?
