# SplitZip: Ultra Fast Lossless KV Compression for Disaggregated LLM Serving

## TL;DR

SplitZip is a GPU-friendly lossless compressor for KV-cache transfer in disaggregated LLM serving, achieving ultra-fast compression and decompression by exploiting redundancy in floating-point exponents.

## Problem & Motivation

In prefill-decode disaggregated LLM serving, prefill workers generate a KV cache that must be transferred to decode workers over the network, becoming a significant bottleneck, especially for long-input and agentic workloads. Existing lossless codecs are not suitable because they target offline weight compression, rely on CPU-side processing, or use variable-length coding that decompresses fast but compresses too slowly for online use.

## Key Ideas

- Exploit redundancy in floating-point exponents of KV activations.
- Encode the most frequent exponent values with fixed-length codes.
- Encode (position, value) pairs and values of rare exponents in an escape stream.
- Use an offline calibrated top-16 exponent codebook for online encoding.
- Combine a regular dense path and sparse escape correction for efficient GPU encoding and decoding.

## System Design

SplitZip operates on GPU. It uses an offline calibrated codebook of the top-16 most frequent exponent values. During encoding, for each KV element, if its exponent is in the codebook, it is encoded with a fixed-length code; otherwise, the exponent and value are encoded as (position, value) pairs in an escape stream. Decoding reconstructs the tensor by combining the dense fixed-length path and the sparse escape correction.

## Evaluation

Evaluated on real BF16 activation tensors. Achieves 613.3 GB/s compression throughput and 2181.8 GB/s decompression throughput, outperforming prior lossless compressors. End-to-end transfer experiments show up to 1.32× speedup for BF16 KV-cache transfer, 1.30× speedup for TTFT, and 1.23× increase in request throughput.

## Takeaways

- Lossless KV-cache compression can significantly reduce transfer latency in disaggregated serving.
- Exploiting exponent redundancy is effective for BF16 tensors.
- Fixed-length coding for frequent exponents enables fast GPU encoding/decoding.
- Offline calibration of codebook is key to online efficiency.

## Related Work

Existing lossless codecs target offline weight compression, rely on CPU, or use variable-length coding (fast decompress but slow compress). SplitZip is designed specifically for online KV-cache transfer on GPU.

## Open Questions

Generalization to other floating-point formats (e.g., FP8) or other data types. Scalability with larger codebooks or adaptive codebooks. Impact on accuracy if lossy compression is allowed.
