# Efficient Memory Management for Large Language Model Serving with PagedAttention

## TL;DR

PagedAttention manages KV cache like virtual memory pages, enabling near-zero waste and flexible sharing. vLLM, built on this idea, achieves 2–4× higher throughput than FasterTransformer and Orca.

## Problem & Motivation

LLM serving requires batching many requests, but each request's KV cache is large and dynamically sized. Existing systems suffer from memory fragmentation and redundant duplication, wasting memory and limiting batch size. This reduces throughput and increases latency.

## Key Ideas

- PagedAttention: treat KV cache as fixed-size blocks (pages) mapped to physical memory via a page table, enabling non-contiguous storage and on-demand allocation.
- Demand-based paging: allocate physical pages only when needed, eliminating internal fragmentation.
- Copy-on-write and sharing: allow multiple requests to share KV cache pages (e.g., for parallel sampling) and copy only on modification.
- Block manager: manages physical page allocation, eviction, and reference counting.

## System Design

vLLM consists of a scheduler, a block manager, and a distributed execution engine. The scheduler batches requests and assigns logical KV cache blocks. The block manager maps logical blocks to physical pages using a page table, handles allocation, eviction (e.g., via FCFS), and reference counting for sharing. The execution engine runs the model with PagedAttention, which uses block-level attention computation. Control flow: scheduler selects requests → block manager allocates pages → engine executes forward pass with PagedAttention → scheduler updates state.

## Evaluation

Evaluated on popular LLMs (e.g., OPT, LLaMA) with workloads from ShareGPT and Alpaca. Baselines: FasterTransformer and Orca. Headline: vLLM improves throughput by 2–4× at same latency. Improvement grows with longer sequences, larger models, and complex decoding (e.g., parallel sampling).

## Takeaways

- OS-inspired paging effectively solves KV cache fragmentation in LLM serving.
- Near-zero memory waste directly translates to higher batch sizes and throughput.
- Sharing KV cache across requests (e.g., beam search) further reduces memory.
- The design is model-agnostic and works with various LLM architectures.

## Related Work

vLLM improves upon prior LLM serving systems (FasterTransformer, Orca) by addressing their memory inefficiency. It is orthogonal to model compression or quantization techniques.

## Open Questions

- How does PagedAttention perform with very long contexts (e.g., 128K tokens) where page table overhead grows?
- Can the block manager be optimized for heterogeneous GPU memory hierarchies (e.g., disaggregated memory)?
- How to extend sharing to more complex decoding strategies (e.g., tree-of-thought)?
