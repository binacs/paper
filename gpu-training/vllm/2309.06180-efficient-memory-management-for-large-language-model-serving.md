# Efficient Memory Management for Large Language Model Serving with PagedAttention

## TL;DR

PagedAttention, inspired by OS virtual memory paging, enables near-zero waste in KV cache memory for LLM serving. Built on this, vLLM achieves 2-4× throughput improvement over state-of-the-art systems like FasterTransformer and Orca.

## Problem & Motivation

LLM serving requires batching many requests for high throughput, but each request's KV cache is large and dynamically sized. Existing systems suffer from significant memory waste due to fragmentation and redundant duplication, limiting batch size and throughput.

## Key Ideas

- PagedAttention: manage KV cache in fixed-size blocks (pages) with a page table, enabling non-contiguous memory allocation and eliminating internal fragmentation.
- Demand-based paging: allocate physical blocks only when needed, reducing waste.
- Copy-on-write: share physical pages across multiple requests (e.g., parallel sampling) to avoid duplication.
- Efficient block management: use a centralized block manager to track free and used blocks.

## System Design

vLLM is a distributed LLM serving system built on PagedAttention. It consists of a scheduler that batches requests, a block manager that allocates and tracks KV cache pages, and an attention kernel that uses page tables to access non-contiguous memory. The scheduler groups requests into iteration-level batches, and the block manager handles page allocation, eviction, and sharing via copy-on-write.

## Evaluation

Evaluated on popular LLMs (e.g., OPT, LLaMA) with workloads from ShareGPT and Alpaca. Baselines: FasterTransformer and Orca. Headline results: vLLM improves throughput by 2-4× under same latency constraints. Improvement is larger for longer sequences, larger models, and complex decoding (e.g., parallel sampling).

## Takeaways

- OS-inspired paging can dramatically reduce memory waste in LLM serving.
- KV cache sharing across requests (e.g., beam search) further reduces memory usage.
- Near-zero waste enables larger batch sizes and higher throughput.
- The approach is orthogonal to model architecture and can be integrated into existing systems.

## Related Work

vLLM improves upon existing LLM serving systems (FasterTransformer, Orca) by addressing memory inefficiency. It is complementary to model parallelism and quantization techniques.

## Open Questions

- How does PagedAttention perform with very long sequences (e.g., 100k+ tokens)?
- Can the page table overhead become significant for extremely large models?
- How to extend to multi-GPU or disaggregated memory settings?
- Potential for combining with speculative decoding or other advanced decoding methods.
