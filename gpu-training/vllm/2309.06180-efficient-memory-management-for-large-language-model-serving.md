# Efficient Memory Management for Large Language Model Serving with PagedAttention

## TL;DR

PagedAttention, inspired by OS virtual memory, manages KV cache in fixed-size blocks to eliminate fragmentation and enable sharing, forming the basis of vLLM, an LLM serving system that achieves 2-4× throughput improvement over FasterTransformer and Orca.

## Problem & Motivation

High-throughput LLM serving requires batching many requests, but each request's KV cache is large and dynamically grows/shrinks during decoding. Existing systems suffer from significant memory waste due to fragmentation and redundant duplication, limiting batch size and throughput.

## Key Ideas

- PagedAttention: attention algorithm that operates on fixed-size blocks (pages) of KV cache, analogous to virtual memory paging.
- Demand-based paging: allocate KV cache blocks on demand as tokens are generated, avoiding pre-allocation waste.
- Block-level memory management: use a block table to map logical blocks to physical blocks, enabling non-contiguous memory allocation and eliminating fragmentation.
- Copy-on-write and sharing: allow multiple requests to share KV cache blocks (e.g., for parallel sampling or beam search) via reference counting, reducing duplication.

## System Design

vLLM is built on PagedAttention. The KV cache is divided into fixed-size blocks (pages). A block table maps logical token positions to physical block IDs. During decoding, new blocks are allocated on demand. For shared prefixes (e.g., in beam search), blocks are shared via copy-on-write. The scheduler manages block allocation and eviction. The system uses a central memory manager to handle block allocation across requests.

## Evaluation

Evaluated on popular LLMs (e.g., OPT, LLaMA) with workloads from ShareGPT and Alpaca. Baselines: FasterTransformer and Orca. Headline: vLLM improves throughput by 2-4× at same latency. Improvement is larger with longer sequences, larger models, and complex decoding (e.g., beam search).

## Takeaways

- OS-inspired paging can dramatically improve memory efficiency in LLM serving.
- Block-level KV cache management eliminates fragmentation and enables sharing.
- vLLM's design is simple yet effective, leading to wide adoption.
- The approach is orthogonal to model architecture and can be integrated into other systems.

## Related Work

vLLM is positioned against existing LLM serving systems like FasterTransformer and Orca, which manage KV cache inefficiently (e.g., pre-allocating maximum size or using list-like structures). It also relates to prior work on attention mechanisms and memory management in deep learning.

## Open Questions

- How does PagedAttention perform with very long contexts (e.g., 100k+ tokens) where block overhead may increase?
- Can the paging idea be extended to other dynamic memory structures in LLMs (e.g., activations)?
- How to optimize block size selection automatically?
- Integration with speculative decoding or other advanced decoding techniques.
