# SpecKV: Adaptive Speculative Decoding with Compression-Aware Gamma Selection

## TL;DR

SpecKV is a lightweight adaptive controller that dynamically selects the speculation length per step in speculative decoding, using draft model confidence and entropy as predictors, achieving 56% improvement over fixed-length baselines with negligible overhead.

## Problem & Motivation

Speculative decoding accelerates LLM inference by having a small draft model propose candidate tokens that a larger target model verifies. A critical hyperparameter is the speculation length γ (number of tokens proposed per step). Existing systems use a fixed γ (typically 4), but the optimal γ varies across task types and depends on the compression level applied to the target model (e.g., FP16, INT8, NF4). There is a need for an adaptive method that selects γ per step to maximize efficiency.

## Key Ideas

- Adaptive speculation length selection per step using a lightweight MLP controller.
- Use of draft model confidence and entropy as strong predictors of acceptance rate (correlation ~0.56).
- Profiling across 4 task categories, 4 speculation lengths, and 3 compression levels to collect 5,112 step-level records.
- Controller maximizes expected tokens per speculation step with <0.5% overhead.

## System Design

SpecKV is a lightweight adaptive controller that sits between the draft model and target model. At each speculation step, it extracts draft model confidence and entropy from the draft model's output. These signals are fed into a small MLP that predicts the optimal γ (speculation length) for that step. The MLP is trained offline on collected profiling data to maximize expected tokens per step. The decision overhead is 0.34 ms, less than 0.5% of step time.

## Evaluation

Evaluated on 4 task categories, 4 speculation lengths, and 3 compression levels (FP16, INT8, NF4). Collected 5,112 step-level records. SpecKV achieves 56.0% improvement over fixed-γ=4 baseline. Statistical significance confirmed with p < 0.001 (paired bootstrap test). Overhead is 0.34 ms per decision.

## Takeaways

- Optimal speculation length varies with task type and target model compression level.
- Draft model confidence and entropy are effective predictors of acceptance rate.
- Adaptive γ selection can significantly improve speculative decoding throughput with minimal overhead.
- Open-source release of profiling data and models facilitates reproducibility.

## Related Work

SpecKV is positioned against existing speculative decoding systems that use a fixed γ (typically 4). It is the first work to adaptively select γ per step using compression-aware signals from the draft model.

## Open Questions

Generalization to other draft/target model pairs beyond those profiled. Impact of different MLP architectures or feature sets. Applicability to other speculative decoding variants (e.g., Medusa, self-speculation).
